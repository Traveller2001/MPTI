# MPTI 项目执行文档

## 1. 项目目标

MPTI 的目标不是“严肃评估导师”，而是做一个站在学生视角、通过日常行为选项来识别导师风格的娱乐型人格测试。

项目要求：

- 保留 `ABTI` 的交互节奏和页面框架
- 将题目视角改为“学生观察导师行为”
- 最终结果保持幽默、轻自嘲、可传播
- 补齐图鉴页、排行榜页和统计能力
- 预留统一的图片生成规范与 Prompt

## 2. 对 ABTI 的拆解结论

从现有 `ABTI` 项目中，抽出了以下可复用骨架：

1. 首页、答题页、结果页三段式结构
2. 单题展示、自动跳下一题、支持上一题/下一题
3. 15 维度、每维 2 题、每题 3 分档
4. 用户向量与标准人格模板之间的曼哈顿距离匹配
5. 当相似度不足时触发 `404` 兜底人格
6. 通过 1 道插入式彩蛋题 + 1 道触发题激活隐藏人格
7. 图鉴页用于展示所有人格卡片
8. 排行榜页通过 Netlify Functions 读取统计摘要
9. 统计逻辑通过 `visit/result` 两类事件累积

## 3. MPTI 的迁移策略

本次迁移没有做“仅改文案”的浅拷贝，而是做了可继续扩展的内容层迁移：

1. 保留原有页面结构与用户操作路径
2. 将人格内容、题库、维度解释抽到 `assets/mpti-data.v20260413a.js`
3. 将首页答题逻辑、图鉴逻辑、排行榜逻辑拆成独立脚本
4. 后端统计函数整体复制一份并切到 `MPTI` 独立存储桶
5. 图片层不强依赖现成素材，先支持缺图占位，再通过 Prompt 批量补图

## 4. 内容体系设计

### 4.1 五大模型

MPTI 延续 5 大模型 × 3 维度 = 15 维结构：

1. 带组管理模型
   - `G1 控制欲`
   - `G2 推进强度`
   - `G3 开会密度`
2. 沟通反馈模型
   - `C1 响应速度`
   - `C2 反馈温度`
   - `C3 画饼浓度`
3. 学术策略模型
   - `A1 选题冒险度`
   - `A2 结果偏好`
   - `A3 资源嗅觉`
4. 培养生态模型
   - `T1 放养程度`
   - `T2 护崽指数`
   - `T3 情绪天气`
5. 导师生存模型
   - `L1 经费手感`
   - `L2 在场概率`
   - `L3 边界感`

### 4.2 题目设计原则

题目全部切换为学生视角，避免直接问“你觉得导师怎样”，而是尽量通过具体行为场景完成判断：

- 看论文会改到什么程度
- 消息回得多快
- 组会有多密
- 遇到投稿/合作冲突时是否护学生
- 经费是抠门还是敢花
- 导师到底在办公室，还是在机场

每个维度保留 2 题，总计 30 题。

### 4.3 人格库设计

标准人格共 24 个，包含但不限于：

- `KING` 王者导师
- `CTRL` 微操总监
- `KPIX` KPI 永动机
- `CAKE` 画饼仙人
- `CASH` 经费喷泉
- `FARM` 放养宗师
- `NANNY` 保姆型导师
- `ADMIN` 行政超导体

特殊人格共 2 个：

- `GHOST` 量子导师：隐藏彩蛋人格
- `404` 查无此导：低相似度兜底人格

## 5. 彩蛋逻辑

保留 ABTI 的插入式彩蛋机制，但改成更贴合导师场景的“神隐检测”：

1. 在正常题目中随机插入“导师最近最常出现在哪里”
2. 如果用户选到“机场/酒店/会议中心”或“主要存在于传说里”
3. 追加一题“你上次和导师单独聊超过 10 分钟是什么时候”
4. 如果回答接近“已经记不清”，直接触发隐藏人格 `GHOST`

这保证了彩蛋人格来自真实观察体验，而不是硬塞梗。

## 6. 图片资产方案

### 6.1 当前实现

当前代码已经支持缺图时的优雅降级：

- 结果页：没有图片时显示人格 code 占位
- 图鉴页：没有图片时显示渐变卡面占位
- 排行榜页：没有图片时显示缩略占位

因此现在就可以开发、测试、上线逻辑，不会被素材卡住。

### 6.2 正式出图规则

- 图片命名：`images/<CODE>/<CODE>.webp`
- 单人格单图
- 统一方图规格
- 全部采用同一角色海报风格

详细 Prompt 见 [IMAGE_PROMPTS.md](./IMAGE_PROMPTS.md)。

## 8. 实施步骤

1. 读取 `ABTI` 页面逻辑和 Netlify 统计机制
2. 重建 `MPTI` 的 15 维框架与 30 道题库
3. 设计 24 个标准人格 + 2 个特殊人格
4. 写入结果文案、维度解释、图鉴标签
5. 接入排行榜所需的独立 analytics store
6. 补充图片命名规范与 Prompt 文档
7. 做静态脚本语法校验

## 9. 后续建议

如果你下一步要继续完善，建议按这个顺序：

1. 先批量生成 `KING`、`CAKE`、`CTRL`、`GHOST` 等辨识度最高的几张图
2. 再根据你身边真实导师风格，微调题目选项的语气
3. 如果后续想更“像 MBTI”，可以继续增加二级结果解释，例如：
   - 最像你的副人格
   - 最危险维度
   - 最适合的应对策略
