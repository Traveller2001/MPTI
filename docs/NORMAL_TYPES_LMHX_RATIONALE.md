# MPTI 四档模板重标说明

## 修改方案

1. 主量表保留原始 `1-4` 答案，不再把 `3/4` 合并成同一个高档。
2. 展示层改为 `L / M / H / X` 四档映射，其中 `X = Extreme`，专门对应题目里的第 4 档极端状态。
3. `normalTypes` 全部重写为 18 维四档模板，顺序固定为 `G1-G2-G3 / C1-C2-C3 / A1-A2-A3 / T1-T2-T3 / L1-L2-L3 / E1-E2-E3`。
4. 匹配距离继续使用逐维绝对差，但最大差值从 `2` 改为 `3`，总分母改成 `18 * 3`。
5. 重标原则不是“把旧 H 随便拆成 H/X”，而是按每个类型描述里的核心人格、伴随特征和相邻类型区分度重新定值。

## 档位定义

- `L = 1`：该维度的低位表现，通常对应题目里的第一档。
- `M = 2`：中位表现，说明该特征存在，但不是主导特征。
- `H = 3`：高位表现，说明该特征很明显，但还没到最极端、最戏剧化的版本。
- `X = 4`：极端高位，只在描述明确指向第四档时使用，用来拉开与普通高位的差异。

## 画像邻近关系

计算规则与页面匹配逻辑一致：

- 仅统计 19 个 `normalTypes` 之间的距离，不含 `WIFI` 和 `NULL`。
- 向量顺序固定为 `G1-G2-G3 / C1-C2-C3 / A1-A2-A3 / T1-T2-T3 / L1-L2-L3 / E1-E2-E3`。
- 距离定义为 18 维逐项绝对差之和，也就是当前代码里的 Manhattan distance。
- 距离越小，说明两个画像在四档模板上越接近；这不代表两个类型相同，只代表它们在量表空间里是最近邻。

## 重标结果

### GOAT 导界之光

- 模板：`MMM-LXL-MLX-MXL-XMX-LLL`
- 区分重点：不是“全高分”导师，而是资源、保护、边界、稳定性同时很强，同时把微操、偏心和压榨压到最低。
- `G组`：`G1=M`，判断强但不接管方向盘；`G2=M`，有要求但不靠高压推进；`G3=M`，会跟进但不把学生泡在会议里。
- `C组`：`C1=L`，不靠比较和操控驱动学生；`C2=X`，能先接住学生再给反馈；`C3=L`，卖点是真托举，不是大饼续命。
- `A组`：`A1=M`，选题稳健有判断；`A2=L`，底线清楚；`A3=X`，资源和合作能力明显属于顶配。
- `T组`：`T1=M`，给框架也给成长空间；`T2=X`，关键时刻真会挡枪；`T3=L`，情绪稳定，不制造气压灾害。
- `L组`：`L1=X`，经费和条件到位；`L2=M`，能找到人但不是全天贴身；`L3=X`，明显尊重作息和边界。
- `E组`：`E1=L`，机会分配基本公平；`E2=L`，署名透明；`E3=L`，不会把学生当劳动力系统来用。

### CTRL 微操狂魔

- 模板：`XHM-MLL-MLM-LMM-MMM-LLM`
- 区分重点：和 `REJ` 的差别在于，CTRL 的核心是“手伸进细节里”，不是单纯逻辑批判。
- `G组`：`G1=X`，会直接改到句子级别；`G2=H`，对推进和完成质量要求都高；`G3=M`，会跟，但不一定高频加会。
- `C组`：`C1=M`，压力主要来自高标准而非精神操控；`C2=L`，反馈偏冷；`C3=L`，不靠宏大叙事动员学生。
- `A组`：`A1=M`，选题并不激进；`A2=L`，对规范通常较严；`A3=M`，资源不是主要标签。
- `T组`：`T1=L`，会手把手改；`T2=M`，不刻意护崽，但也不一定甩锅；`T3=M`，情绪相对可预测。
- `L组`：`L1=M`，经费中性；`L2=M`，能约到；`L3=M`，边界感普通。
- `E组`：`E1=L`，偏心不是主问题；`E2=L`，署名通常还按规矩来；`E3=M`，会大量占用你时间做精修，但不等于劳动力榨取。

### PUSH 催命判官

- 模板：`HXH-MLM-HMH-MMM-MML-LLM`
- 区分重点：和 `996` 的差别在于，PUSH 是把你往主线产出上压，不是把你拖去做一堆无关杂活。
- `G组`：`G1=H`，会对任务和节奏有明显掌控；`G2=X`，推进强度是核心标签；`G3=H`，跟进频率也高。
- `C组`：`C1=M`，会施压但主要对事；`C2=L`，沟通温度偏低；`C3=M`，会给希望但不是饼王。
- `A组`：`A1=H`，凡是可能出成果的方向都敢推；`A2=M`，底线通常正常，但效率优先感很强；`A3=H`，通常手里机会和项目不算少。
- `T组`：`T1=M`，会给方向，但默认你自己扛着跑；`T2=M`，有限度缓冲；`T3=M`，更像稳定高压，不是情绪风暴。
- `L组`：`L1=M`，资源够推进；`L2=M`，还能约到；`L3=L`，边界感弱，DDL 常压过你的私人安排。
- `E组`：`E1=L`，不一定偏心；`E2=L`，署名不是核心问题；`E3=M`，会严重占用时间，但仍主要围绕课题产出。

### CAKE 画饼大仙

- 模板：`MHM-MHX-HMM-MMM-MMH-LLL`
- 区分重点：和 `YOLO` 的差别在于，CAKE 的主导能力是“放大叙事”，不是“真实押风险”。
- `G组`：`G1=M`，更多是语言带方向，不是细节接管；`G2=H`，会把目标说得很大；`G3=M`，跟进频率正常。
- `C组`：`C1=M`，会借情绪和话术调动人；`C2=H`，反馈里有明显鼓舞感；`C3=X`，画饼是核心特征。
- `A组`：`A1=H`，会把题目往更大空间讲；`A2=M`，通常不越底线；`A3=M`，资源不是最大优势。
- `T组`：`T1=M`，给方向但不代跑；`T2=M`，护崽一般；`T3=M`，气压相对稳定。
- `L组`：`L1=M`，经费普通；`L2=M`，可约；`L3=H`，边界通常还比较体面。
- `E组`：`E1=L`，偏心不是主问题；`E2=L`，署名不是主问题；`E3=L`，问题在叙事，不在制度性剥削。

### ATM 行走经费机

- 模板：`MHM-LMM-HLX-MML-XHM-MLM`
- 区分重点：和 `GOAT` 的差别在于，ATM 强在搞资源，不等于在温度、保护和边界上也同样封神。
- `G组`：`G1=M`，主要控制资源配置而不是写作细节；`G2=H`，机会多，推进感也更强；`G3=M`，跟进正常。
- `C组`：`C1=L`，不靠操控；`C2=M`，沟通偏务实；`C3=M`，会讲机会，但不是纯靠饼推进。
- `A组`：`A1=H`，因为有资源所以敢开更大的方向；`A2=L`，资源强不等于底线差；`A3=X`，资源嗅觉是核心。
- `T组`：`T1=M`，能给路线但不是保姆型；`T2=M`，保护性普通；`T3=L`，情绪稳定。
- `L组`：`L1=X`，经费手感拉满；`L2=H`，资源型导师往往更忙、更难约；`L3=M`，边界感正常。
- `E组`：`E1=M`，资源分配可能更偏向能打的人；`E2=L`，署名通常还算清楚；`E3=M`，机会多也会把学生卷得更忙。

### ZOOM 组会永动机

- 模板：`HHX-LML-MLM-MMM-MLM-LLM`
- 区分重点：和 `5G` 的差别在于，ZOOM 是同步场景高密度占满，不只是消息回得快。
- `G组`：`G1=H`，高频跟进本身就是控制；`G2=H`，会议服务于推进；`G3=X`，组会密度就是核心特征。
- `C组`：`C1=L`，对事推进，不靠情绪操控；`C2=M`，反馈温度普通；`C3=L`，不靠大饼。
- `A组`：`A1=M`，选题策略不极端；`A2=L`，规范通常正常；`A3=M`，资源中位。
- `T组`：`T1=M`，不是放养，问题会被拉进会议拆；`T2=M`，会有限度兜一下；`T3=M`，情绪并非主变量。
- `L组`：`L1=M`，经费中性；`L2=L`，线下实体化率高；`L3=M`，边界感一般，随机会随时打断你。
- `E组`：`E1=L`，偏心不是主标签；`E2=L`，署名问题不突出；`E3=M`，时间会被会议吞掉，但主线还是课题。

### AFK 放养宗师

- 模板：`LLL-LML-MML-XLL-MXX-LLL`
- 区分重点：和 `ZEN` 的差别在于，AFK 不是温和放手，而是存在感和实际支持都更低。
- `G组`：`G1=L`，不控细节；`G2=L`，不催进度；`G3=L`，几乎不主动组织高频跟进。
- `C组`：`C1=L`，不靠操控；`C2=M`，不算冷酷，但也谈不上接住情绪；`C3=L`，不靠愿景续命。
- `A组`：`A1=M`，有时会让你自己试；`A2=M`，底线普通，但缺少强监督；`A3=L`，资源铺设能力不突出。
- `T组`：`T1=X`，求助也多半靠你自己悟；`T2=L`，外部压力来了主要自己扛；`T3=L`，气压稳定，因为人都不太出现。
- `L组`：`L1=M`，经费够活但不奢侈；`L2=X`，极难观测到实体；`L3=X`，边界感很强，几乎不打扰你。
- `E组`：`E1=L`，偏心不是主问题；`E2=L`，署名不是主问题；`E3=L`，问题在缺席，不在掠夺。

### GPS 全能保姆

- 模板：`MMM-LXL-LLM-LHL-MLH-LLL`
- 区分重点：和 `TANK` 的差别在于，GPS 的强项是日常照料和补位，不是危机时刻冲出去挡枪。
- `G组`：`G1=M`，会管很多实际细节；`G2=M`，推进有但不高压；`G3=M`，跟进频率正常。
- `C组`：`C1=L`，不靠比较压人；`C2=X`，反馈温度非常高；`C3=L`，风格务实，不靠大饼。
- `A组`：`A1=L`，选题偏稳；`A2=L`，规范意识强；`A3=M`，资源够用但不是顶配型。
- `T组`：`T1=L`，求助时会一起拆问题；`T2=H`，会站学生一边，但没到 TANK 那种重装护崽；`T3=L`，情绪稳定。
- `L组`：`L1=M`，经费够用；`L2=L`，导师比较在场；`L3=H`，基本知道尊重你的休息时间。
- `E组`：`E1=L`，机会相对公平；`E2=L`，署名透明；`E3=L`，学生还是按学生来带。

### TANK 护崽坦克

- 模板：`MMM-LHL-MLM-MXL-MMH-LLL`
- 区分重点：和 `GPS` 的差别在于，TANK 的核心是“对外吸收伤害”，不是日常生活流照顾。

### 5G 秒回战神

- 模板：`MHH-LML-MLM-MML-MML-LLM`
- 区分重点：和 `007` 的差别在于，5G 的主特征是“响应速度快”，不是“默认你 24 小时待命”。
- `G组`：`G1=M`，控制更多通过高响应建立；`G2=H`，节奏快；`G3=H`，线上跟进频率高。

- `A组`：`A1=M`，选题策略中位；`A2=L`，规范通常在线；`A3=M`，资源普通。
- `T组`：`T1=M`，会迅速给方向；`T2=M`，能及时接住但不是护崽坦克；`T3=L`，气压稳定。

- `E组`：`E1=L`，机会分配一般不算偏；`E2=L`，署名通常正常；`E3=M`，持续在线会显著侵占你的时间。

### YOLO 选题赌神

- 模板：`MHM-LMH-XMH-MMM-MMM-LLM`
- 区分重点：和 `CAKE` 的差别在于，YOLO 真会把筹码押到高风险方向上，不只是把话说大。
